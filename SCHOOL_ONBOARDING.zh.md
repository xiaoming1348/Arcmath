# 学校接入手册（学校侧）

这是 ArcMath 试点学校的"第一天"指南，按三个角色分写：
**管理员**（每校一人）、**老师**、**学生**。

> 跟 `PILOT_ONBOARDING.zh.md` 的区别：那份是 ArcMath 运营对接学校的
> 内部 runbook；这份是给学校直接读的，从你拿到 admin 账号那一刻
> 开始算起。

试点配额：每校 1 位管理员、最多 5 位老师、最多 50 位学生。

---

## 一、学校管理员（admin）

### 1.1 你会从我们这里收到什么

ArcMath 给你两条信息（通过安全渠道，不会和登录链接走同一封邮件）：

- **登录地址**：`https://<your-pilot-domain>/login`
- **管理员账号**：邮箱形式，例如 `admin@<your-school>.arcmath.local`
- **临时密码**：16 位随机字符串

第一次登录后立即修改密码（`/login/set-password` 会自动弹）。

### 1.2 创建班级 + 录入老师/学生（一次性）

**这是和很多其他平台不一样的地方**：你不需要让老师/学生先各自注册再
拉进来。你直接录名字，系统就**自动建账号**给他们。

打开 `/org`（管理员总览），点「创建班级」：

1. **班级名称**：例如「十年级数学冲刺班 2026」
2. **指派老师**：每行一位
   - 「新建」→ 输入中文/英文姓名（中文会被转成拼音作为 email 前缀）
   - 「已存在」→ 从你之前录过的老师里挑（适合一位老师带多个班）
3. **学生名单**：每行一位
   - 同样支持「新建」和「已存在」
   - 中文姓名 → 拼音 slug + 4 位随机后缀，例如
     `zhangsan.7a3f@<school-slug>.arcmath.local`
   - 一次最多录 50 个，超过就分批

提交后系统会**当场**生成所有账号 + 临时密码，并把凭据列表展示给你。
**复制保存**，因为关闭页面就看不到了。

### 1.3 把账号发给老师和学生

把每个老师/学生的 `(email, 临时密码)` 通过你们学校自己的渠道发给本人
（钉钉、企业微信、纸条都行）。**别在公共群里发** —— 这是临时密码，
对方第一次登录会被强制改成自己的密码。

如果有人忘了临时密码，你在 `/org` 上能看到那一行有「重置密码」按钮，
点了会生成新临时密码，再发给他/她。

### 1.4 管理面板能看什么

- **`/org`**：所有班级、老师、学生在这里
- **`/org/students/<id>`**：单个学生的答题历史
- **`/resources`**：上传课程资料（PDF / 文本说明），全校老师学生可见
- 各角色独立的着陆页：`/teacher`（老师）、`/dashboard`（学生）

## 二、老师（teacher）

### 2.1 第一次登录

1. 用学校管理员发的邮箱 + 临时密码登录 `/login`
2. 系统自动跳到 `/login/set-password`，设你自己的密码（≥10 位）
3. 跳到 `/teacher`，能看到管理员已经分给你的班

### 2.2 上传一份作业

ArcMath 默认有个题集库（约 74 套真题，AMC8/10/12 + AIME + Euclid +
MAT + STEP + USAMO + Putnam）。你也可以传自己的。

**两种路径：**

**（A）从题集库挑现成的**

1. 进 `/teacher/classes/<班级 ID>`
2. 「布置作业」→ 选题集（可按比赛、年份筛）→ 选标题、起止日期 → 提交

**（B）上传自己的题集**

1. 进 `/teacher/upload`
2. 把题集做成一个 JSON 文件（[teacher-v1 schema](/Users/yimingsun/Desktop/Arcmath/packages/shared/src/teacher-problem-set-schema.ts)，参考 `apps/web/src/scripts/e2e-core-flow.ts` 里的示例）
3. 上传 → 系统会先 dry-run 校验（每题必须有题干 + 答案 / MC 选项 / 证明大纲三选一），通过才入库
4. 题集入库后回到 `/teacher/classes/<班级 ID>` 布置

学生看不到 dry-run 报错，他们只能看到你已经布置的、通过校验的作业。

### 2.3 看进度

`/teacher/classes/<班级 ID>` 里每个作业都有一个进度面板：
- 班里多少人开始 / 已交 / 正确率
- 点单个学生 → 看他每题的提交、提示请求、正确与否

学生写的步骤（不是只填最终答案）也能看到，包括 SymPy / Lean 给的
单步验证结果。

## 三、学生（student）

### 3.1 第一次登录

1. 用老师/管理员给的邮箱 + 临时密码登录 `/login`
2. 系统跳 `/login/set-password`，自己设新密码
3. 跳 `/dashboard`，能看到老师布置的作业

### 3.2 答一道题

点任意作业 → 题集页面 → 点单题 → 进 `/problems/<id>` 解题工作台。

工作台默认显示三个选择：

| 选项 | 适合什么时候用 |
|---|---|
| **我已经做出来了** | 草稿纸上做完了，想直接交答案 |
| **试了但卡住了** | 写了几步但卡住，想让系统逐步检查 |
| **完全没头绪** | 让系统逐级给提示（最多 3 级） |

挑一种就开始。三种之间随时可切换 —— 比如先要提示，提示完想自己写
步骤，点「我来试试写步骤」就行。

#### 自动批改

- **选择题（AMC）**：选 A/B/C/D/E，提交后立刻知道对错 + 简短讲解
- **填空题（AIME / Euclid 短题）**：直接填整数，系统会吃掉 `+`、空格、
  逗号等格式噪音
- **证明 / 步骤题（USAMO / Putnam / STEP / MAT 长题）**：写完官方解答
  会展开给你对照（不自动判分，因为高级题需要看完整证明）

### 3.3 提示流（HINT_GUIDED）

最多 3 级，越后越具体：
- **提示 1**：方向性的（"考虑用 Pythagorean 定理" / "对称化代换"）
- **提示 2**：具体的关键观察（"先求 $\angle ABC = 30°$"）
- **提示 3**：完整解题大纲（最后一根稻草）

请求提示不会扣分，但用过几个提示老师能在进度面板里看到。

### 3.4 切换语言

右上角有个 EN/中文 切换。**默认全员英文**，因为大部分材料是英文的。
切到中文后界面变中文，但题干还是原文（题目是英文比赛的就还是英文）。

## 四、常见问题

**Q：临时密码忘了怎么办？**
A：找学校管理员，他能在 `/org` 里给你重置。

**Q：老师能改学生的答题记录吗？**
A：不能。老师只读，但能看到学生用过的提示数和提交时间。

**Q：学校管理员能看学生的密码吗？**
A：不能。重置密码生成的是新的临时密码，老的就废了。

**Q：试点完了数据怎么办？**
A：默认保留，方便对比第二期。要清空可以联系 ArcMath 运营。

**Q：能离线用吗？**
A：不能。需要联网，因为提示和单步验证走的是 OpenAI + 我们自己的
Python 验证服务。
